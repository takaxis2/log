
## 요약

이 문서는 데이터 과학 및 AI의 기초가 되는 핵심 수학 개념인 수열, 순열, 조합에 대한 포괄적인 분석을 제공한다. 주요 내용은 세종사이버대학교 김명배 교수의 강의 자료를 바탕으로 구성되었다.

수열은 규칙성을 가진 숫자의 나열로 정의되며, 연속된 항의 차이가 일정한 **등차수열**과 비율이 일정한 **등비수열**로 구분된다. 수열의 항이 무한히 커질 때 특정 값에 가까워지는 현상을 **수렴**, 그렇지 않은 경우를 **발산**이라 하며, 이는 수열의 장기적인 행태를 분석하는 데 중요한 개념이다.

경우의 수를 계산하는 방법론인 순열과 조합은 '순서'의 중요성을 기준으로 나뉜다. **순열(Permutation)**은 서로 다른 n개에서 r개를 선택하여 '일렬로 나열'하는 경우의 수로, 순서가 중요하다. 반면, **조합(Combination)**은 순서를 고려하지 않고 단순히 '선택'하는 경우의 수이다. 두 개념 모두 중복을 허용하는지에 따라 **중복순열**과 **중복조합**으로 확장될 수 있다.

이러한 개념들은 파이썬의 `itertools` 라이브러리를 통해 효율적으로 구현될 수 있으며, 알고리즘 및 데이터 분석의 다양한 문제 해결에 필수적인 도구로 활용된다.

## 1. 수열의 이해

수열은 데이터 과학에서 패턴을 인식하고 예측 모델을 구축하는 데 기초가 되는 개념이다.

### 1.1. 개념 정의

- **수열(Sequence)**: 규칙성이 있는 숫자의 나열. 수학적으로는 자연수를 정의역으로 하는 함수 또는 그 결과로 얻어지는 원소들의 집합으로 정의된다.
- **항(Term)**: 수열을 구성하는 각 숫자를 의미하며, 순서에 따라 첫째 항(제1항), 둘째 항(제2항) 등으로 부른다.
- **일반항(General Term)**: 수열의 n번째 항을 나타내는 식으로, `𝑎𝑛`으로 표기한다. 예를 들어, 짝수의 수열 `2, 4, 6, ...`의 일반항은 `𝑎𝑛 = 2𝑛`이다.

### 1.2. 주요 수열 유형

#### 등차수열 (Arithmetic Sequence)

- **정의**: 연속하는 두 항의 차이가 모두 일정한 수열.
- **공차(Common Difference)**: 이웃한 두 항의 일정한 차이를 의미하며 `d`로 표기한다.
- **일반항 공식**: `𝑎𝑛 = 𝑎 + (𝑛 − 1)𝑑` (여기서 `a`는 첫째 항, `d`는 공차)
- **예시**: 수열 `{1, 3, 5, 7, ...}`은 첫째 항이 1이고 공차가 2인 등차수열이며, 일반항은 `𝑎𝑛 = 1 + (𝑛 − 1) × 2 = 2𝑛 − 1`이다.

#### 등비수열 (Geometric Sequence)

- **정의**: 각 항이 그 이전 항에 일정한 수를 곱하여 얻어지는 수열. (단, 첫째 항은 0이 아님)
- **공비(Common Ratio)**: 곱해지는 일정한 수를 의미하며 `r`로 표기한다.
- **일반항 공식**: `𝑎𝑛 = 𝑎𝑟^(𝑛−1)` (여기서 `a`는 첫째 항, `r`은 공비)
- **예시**: 수열 `{2, 4, 8, 16, ...}`은 첫째 항이 2이고 공비가 2인 등비수열이며, 일반항은 `𝑎𝑛 = 2 × 2^(𝑛−1) = 2^𝑛`이다.

## 2. 수열의 극한과 행태 분석

수열의 극한은 n이 무한히 커질 때 수열의 값이 어떻게 변하는지를 분석하는 개념이다.

### 2.1. 수렴과 발산

- **수렴(Convergence)**: 수열 `{𝑎𝑛}`에서 `n`이 무한히 커질 때, `𝑎𝑛`의 값이 일정한 값 `α`에 한없이 가까워지는 상태. 이때 `α`를 수열의 **극한값**이라 한다.
    - 표기: `lim(𝑛→∞) 𝑎𝑛 = α`
- **발산(Divergence)**: 수열이 어떤 특정 값으로 수렴하지 않는 상태. 발산의 종류는 다음과 같다.
    - **양의 무한대로 발산**: `lim(𝑛→∞) 𝑎𝑛 = ∞`
    - **음의 무한대로 발산**: `lim(𝑛→∞) 𝑎𝑛 = −∞`
    - **진동(Oscillation)**: 양의 무한대나 음의 무한대로 발산하지 않으면서 일정한 값에 수렴하지도 않는 경우 (예: `{-1, 1, -1, 1, ...}`).

### 2.2. 등비수열의 수렴 및 발산 조건

등비수열 `𝑎𝑛 = 𝑎𝑟^(𝑛−1)`의 수렴과 발산은 첫째 항 `a`와 공비 `r`의 값에 따라 결정된다.

- **a = 0 인 경우**: 모든 항이 0이므로 `0`으로 수렴한다.
- **a ≠ 0 인 경우**:
    - `r > 1`: 양의 무한대로 발산 (`∞`).
    - `r = 1`: 첫째 항 `a`로 수렴한다.
    - `−1 < r < 1`: `0`으로 수렴한다.
    - `r = −1`: `a, -a, a, -a, ...`로 진동하며 발산한다.
    - `r < −1`: 부호가 바뀌며 절댓값이 커지는 형태로 진동하며 발산한다.

### 2.3. 수열 극한의 기본 성질

두 수열 `{𝑎𝑛}`과 `{𝑏𝑛}`이 각각 `α`와 `β`로 수렴할 때, 다음과 같은 성질이 성립한다.

1. `lim(𝑛→∞) 𝑐𝑎𝑛 = 𝑐α` (단, `c`는 상수)
2. `lim(𝑛→∞) (𝑎𝑛 + 𝑏𝑛) = α + β`
3. `lim(𝑛→∞) (𝑎𝑛 − 𝑏𝑛) = α − β`
4. `lim(𝑛→∞) (𝑎𝑛𝑏𝑛) = αβ`
5. `lim(𝑛→∞) (𝑎𝑛 / 𝑏𝑛) = α / β` (단, `𝑏𝑛 ≠ 0`, `β ≠ 0`)

## 3. 순열: 순서가 있는 배열

순열은 선택된 항목들의 순서나 위치가 결과에 영향을 미치는 경우의 수를 계산하는 데 사용된다.

### 3.1. 기본 순열 (Permutation)

- **정의**: 서로 다른 `n`개에서 `r`개를 선택하여 일렬로 나열하는 것. **순서가 중요**하고 **중복은 허용되지 않는다**.
- **표현**: `nPr` 또는 `P(n, r)`
- **계산 공식**: `nPr = n! / (n-r)! = n × (n-1) × ... × (n-r+1)`
- **주요 성질**:
    - `n P n = n!`
    - `n P 0 = 1`
    - `0! = 1` (정의에 따라)
- **파이썬 구현**: `itertools` 라이브러리의 `permutation()` 함수를 사용한다.

### 3.2. 중복 순열 (Permutation with Repetition)

- **정의**: 서로 다른 `n`개에서 **중복을 허락하여** `r`개를 선택하고 일렬로 나열하는 것.
- **표현**: `nΠr`
- **계산 공식**: `nΠr = n^r`
- **예시**: 숫자 `{1, 2, 3, 4, 5, 6}`을 사용하여 만들 수 있는 세 자리 자연수의 경우
    - 중복을 허락하지 않는 경우(순열): `6P3 = 6 × 5 × 4 = 120`
    - 중복을 허락하는 경우(중복순열): `6Π3 = 6³ = 216`

## 4. 조합: 순서 없는 선택

조합은 전체 집합에서 일부를 선택할 때 순서에 관계없이 구성 자체가 중요한 경우의 수를 계산하는 데 사용된다.

### 4.1. 기본 조합 (Combination)

- **정의**: 서로 다른 `n`개에서 **순서를 생각하지 않고** `r`개를 선택하는 것. **순서가 중요하지 않고** **중복은 허용되지 않는다**.
- **표현**: `nCr` 또는 `C(n, r)`
- **계산 공식**: `nCr = nPr / r! = n! / (r! × (n-r)!)`
- **파이썬 구현**: `itertools` 라이브러리의 `combinations()` 함수를 사용한다.
- **예시**: 40명의 학생 중 반장 1명, 부반장 1명을 뽑는 경우는 순서가 중요하므로 순열(`40P2`), 주번 3명을 뽑는 경우는 순서가 중요하지 않으므로 조합(`40C3`)을 사용한다.

### 4.2. 중복 조합 (Combination with Repetition)

- **정의**: 서로 다른 `n`개에서 **중복을 허락하여** `r`개를 선택하는 것.
- **표현**: `nHr`
- **계산 공식**: `nHr = (n+r-1)Cr`
- **예시**: 네 종류의 공에서 중복을 허용하여 일곱 개를 택하는 경우의 수는 `4H7 = (4+7-1)C7 = 10C7 = 10C3 = 120`이다.

## 5. 순열과 조합의 핵심 비교 분석

순열과 조합의 가장 근본적인 차이는 '순서와 위치'의 중요성 여부이다. 순열은 '배열하다'의 개념에, 조합은 '뽑는다'의 개념에 가깝다. 이를 중복 허용 여부와 함께 정리하면 다음과 같다.

|   |   |   |   |
|---|---|---|---|
|구분|순서|중복|표현 (Notation)|
|**순열**|있음|불가능|`nPr`|
|**중복순열**|있음|가능|`nΠr = n^r`|
|**조합**|없음|불가능|`nCr`|
|**중복조합**|없음|가능|`nHr = (n+r-1)Cr`|
