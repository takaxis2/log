
## 요약 (Executive Summary)

본 문서는 운영체제의 핵심 구성 요소인 파일 시스템, 디스크 공간 관리, 그리고 고성능 컴퓨팅을 위한 병렬, 다중, 분산 처리 시스템에 대한 심층 분석을 제공한다. 주요 내용은 다음과 같다.

- **파일 시스템 관리:** 파일 시스템은 운영체제가 보조기억장치에 저장된 데이터를 효율적으로 관리하고 접근하기 위해 제공하는 핵심 모듈이다. 이는 파일의 정의, 디렉토리 구조, 파일 연산 등을 포함하며, 사용자에게는 논리적 관점의 인터페이스를 제공하고 물리적 저장 장치를 관리하는 역할을 수행한다. 파일 디스크립터(FCB)는 파일 접근 시 운영체제가 필요로 하는 모든 정보를 담고 있는 자료 구조이다.
- **디렉토리 및 디스크 공간 관리:** 디렉토리는 파일의 위치 정보를 제공하는 심볼 테이블 역할을 하며, 단일 구조, 2단계 구조, 계층적 구조, 비순환 그래프 구조 등 다양한 형태로 구성된다. 한정된 디스크 공간을 효율적으로 관리하기 위해 비트 벡터, 연결 리스트 등의 기법으로 가용 공간을 추적하고, 연속 할당, 연결 할당, 인덱스 할당 방식으로 파일에 공간을 할당한다.
- **고급 처리 시스템 아키텍처:** 병렬, 다중, 분산 처리 시스템은 모두 여러 처리기를 동시에 사용하지만 목적과 구조에서 뚜렷한 차이를 보인다.
    - **병렬 처리(Parallel Processing)**는 단일 CPU 내의 여러 처리 요소(PE)가 하나의 프로그램을 협력하여 처리함으로써 프로그램의 실행 속도 향상을 목표로 한다.
    - **다중 처리(Multi Processing)**는 여러 개의 프로그램을 동시에 병렬로 처리하여 시스템 전체의 처리율(Throughput) 향상을 목표로 한다.
    - **분산 처리(Distributed Processing)**는 네트워크로 연결된 다수의 컴퓨터가 자원과 정보를 공유하며 협력적으로 작업을 처리하여 신뢰성과 효율성을 높인다.
- **다중 처리 시스템의 분류:** 다중 처리 시스템은 다양한 기준에 따라 분류된다. Flynn의 분류법은 명령어와 데이터 흐름에 따라 SISD, SIMD, MISD, MIMD로 구분하며, 이 중 다중 명령어와 다중 데이터 흐름을 갖는 MIMD가 진정한 병렬 처리 구조로 간주된다. 이 외에도 메모리 결합도(강결합/약결합), 프로세서 연결 방식(공유 버스, 크로스바 교환 행렬), 운영체제 형태(주/종, 분리 수행, 대칭적 처리) 등에 따라 세분화된다.

## 1. 파일 시스템 관리

파일 시스템은 사용자가 보조기억장치에 저장된 데이터와 프로그램을 효율적으로 관리하고 접근할 수 있도록 운영체제가 제공하는 핵심적인 모듈이다.

### 1.1. 파일 및 파일 시스템의 개념

- **파일(File):** 연관된 데이터들의 집합에 고유한 이름을 부여한 것으로, 디스크나 테이프 같은 보조기억장치에 저장된다.
- **파일 시스템(File System):** 파일들을 효율적으로 저장하고 접근하기 위한 운영체제의 모듈이다. 사용자가 가장 가깝게 접하는 운영체제 요소이며, 파일의 정의, 내부 구조, 디렉토리 구조, 파일 연산 등의 개념을 포함한다.
- **파일 시스템의 기능:**
    - **사용자 인터페이스 제공:** 사용자에게 파일 입출력 명령과 같은 논리적 관점을 제공하여 사용 편의성을 높인다.
    - **물리적 장치 관리:** 디스크 스케줄링, 블록 전송, 오류 검출 등 물리적 관점의 하위 기능을 수행하여 신뢰성과 효율성을 보장한다.
    - **자료 관리 및 보호:** 디렉토리 구조 제공, 디스크 공간 할당 및 회수, 파일 보호, 데이터 무결성(백업), 암호화 등 상위 기능을 통해 데이터를 체계적으로 관리하고 보호한다.

### 1.2. 파일 디스크립터 (File Descriptor)

파일 디스크립터는 파일 제어 블록(File Control Block, FCB)이라고도 불리며, 파일이 접근되는 동안 운영체제가 해당 파일을 관리하기 위해 필요한 모든 정보를 모아 놓은 자료 구조이다. 파일이 열리면(Open) 해당 파일의 FCB가 메모리에 적재된다.

**파일 디스크립터에 포함되는 정보:**

- 파일 ID, 이름, 디스크 내 주소
- 파일 크기, 생성/수정/최종 접근 시간
- 파일 구조 (순차, 직접 접근 등) 및 파일 종류 (데이터, 실행 파일 등)
- 보호 관련 정보 (쓰기 가능, 실행 가능 등)
- 현재 접근 중인 위치 및 파일을 사용 중인 프로세스 수

### 1.3. 파일 연산 및 구성

#### 파일 연산

- **파일 단위 작업:** `Create`(생성), `Destroy`(삭제), `Open`(열기), `Close`(닫기), `Copy`(복사), `Rename`(이름 변경), `List`(내용 출력)
- **파일 내 데이터 항목 단위 작업:** `Read`(읽기), `Write`(쓰기), `Update`(수정), `Insert`(삽입), `Delete`(삭제)

#### 파일 구성 방법

- **수행 기능에 따른 분류:**
    - **마스터 파일 (Master File):** 조직체의 모든 면을 나타내는 핵심 데이터 파일.
    - **트랜잭션 파일 (Transaction File):** 마스터 파일의 내용을 변경시키는 내역을 담은 파일.
    - **보고 파일 (Report File):** 사용자에게 보여주기 위한 특정 양식을 갖춘 파일.
    - **작업 파일 (Work File):** 작업 중간에 생성되어 다른 프로그램의 입력으로 사용되는 임시 파일.
    - **프로그램 파일 (Program File):** 프로그래밍 언어로 작성된 명령문을 포함하는 파일.
- **파일 구조에 따른 분류:**
    - **순차 파일 (Sequence File):** 레코드가 물리적인 순서대로 저장된 파일.
    - **직접 파일 (Direct File):** 레코드 키(Key)를 주소로 변환하여 직접 접근하는 파일.
    - **인덱스된 순차 파일 (Indexed Sequential File):** 인덱스(색인)를 통해 임의 접근과 순차 접근이 모두 가능한 파일.
    - **다중 링 파일 (Multiple Ring File):** 포인터를 통해 여러 종류의 레코드를 연결하여 구성한 파일.

### 1.4. 파일 접근 방식 비교

파일의 데이터 레코드가 디스크에 저장되는 방식에 따라 접근 방식이 결정된다.

|   |   |   |   |
|---|---|---|---|
|구분|순차 파일 (Sequential File)|인덱스된 순차 파일 (Indexed Sequential File)|직접 파일 (Direct File)|
|**접근 방식**|순차적 접근|색인된 순차적 접근|직접 접근|
|**키(Key) 유무**|없음|있음|있음|
|**주요 기억 매체**|자기 테이프, 디스크|디스크, 드럼|디스크, 드럼|
|**용도**|저장용, 일괄 처리, 대량 파일|일괄 및 대화형 처리|대화형, 자주 바뀌는 소량 파일|
|**장점**|구조 간단, 버퍼링 용이|파일 수정 용이, 비교적 빠른 접근|매우 빠른 접근, 수정 용이|
|**단점**|접근 속도 매우 느림, 수정 어려움|키 필요, 주기적 재구성 필요|주소 계산 시간 필요, 가변 길이 레코드 처리 어려움|

## 2. 디렉토리 및 디스크 공간 관리

### 2.1. 디렉토리 구조

디렉토리는 파일 이름을 입력받아 해당 파일의 위치를 알려주는 심볼 테이블(Symbol Table) 역할을 한다. 디렉토리는 파일 이름, 위치 주소, 크기, 유형, 생성/수정 시기 등 파일에 대한 메타데이터를 포함한다.

- **단일 디렉토리 구조:** 모든 파일이 하나의 디렉토리 안에 위치하는 가장 단순한 구조.
- **2단계 디렉토리 구조:** 마스터 디렉토리 아래에 각 사용자별 디렉토리를 두는 구조.
- **계층적(트리) 디렉토리 구조:** 하나의 루트 디렉토리 아래에 여러 하위 디렉토리가 트리 형태로 구성되는 구조. (예: UNIX, Windows)
- **비순환 그래프 디렉토리 구조:** 여러 디렉토리가 하나의 파일이나 하위 디렉토리를 공유할 수 있는 구조.
- **일반 그래프 디렉토리 구조:** 순환(Cycle)을 허용하는 가장 유연하지만 복잡한 구조.

### 2.2. 디스크 공간 관리

파일 시스템은 한정된 보조기억장치 공간을 효율적으로 관리해야 한다.

#### 디스크 가용 공간 관리 기법

- **비트 벡터 (Bit Vector):** 디스크의 각 블록을 비트(bit) 하나에 대응시켜, 가용 블록은 1, 사용 중인 블록은 0으로 표시하는 기법.
- **연결 리스트 (Linked List):** 모든 가용 디스크 블록을 포인터로 연결하여 관리하는 기법.
- **그룹핑 (Grouping):** 첫 번째 가용 블록에 여러 개의 다른 가용 블록 주소를 저장하여 관리하는 기법.
- **카운팅 (Counting):** 연속된 가용 블록들의 첫 번째 블록 주소와 연속된 블록의 수를 기록하여 관리하는 기법.

#### 디스크 공간 할당 기법

- **연속 할당 (Contiguous Allocation):** 파일을 디스크의 연속된 블록에 할당하는 방식. 접근 속도는 빠르지만 외부 단편화가 발생할 수 있다.
- **연결 할당 (Linked Allocation):** 각 블록이 다음 블록을 가리키는 포인터를 포함하는 방식. 외부 단편화는 없지만 직접 접근이 어렵고 신뢰성이 낮다.
- **인덱스 할당 (Indexed Allocation):** 파일의 모든 데이터 블록 주소를 인덱스 블록 하나에 모아 관리하는 방식. 직접 접근이 가능하고 외부 단편화도 없지만, 인덱스 블록을 위한 추가 공간이 필요하다.

## 3. 병렬, 다중 및 분산 처리 시스템

이 시스템들은 여러 처리기를 동시에 사용한다는 공통점이 있지만, 구조와 목적에서 명확히 구분된다.

### 3.1. 시스템 개요 및 핵심 차이점

- **병렬 처리 (Parallel Processing):** 하나의 CPU 내에 여러 처리 요소(Processing Element, PE)를 두어, 하나의 프로그램을 여러 부분으로 나누어 동시에 처리한다. **목표는 단일 프로그램의 처리 속도를 높이는 것**이다.
- **다중 처리 (Multi Processing):** 여러 개의 독립된 CPU를 사용하여 여러 개의 프로그램을 동시에 병렬로 처리한다. **목표는 여러 프로그램을 동시에 처리하여 시스템 전체의 처리율(Throughput)을 향상시키는 것**이다.
- **분산 처리 (Distributed Processing):** 네트워크로 연결된 다수의 독립적인 컴퓨터들이 자원과 정보를 공유하며 협력적으로 작업을 처리하는 환경이다. **목표는 자원 공유, 신뢰성 향상, 작업 효율 증대**이다.

### 3.2. 다중 처리 시스템 상세 분석

#### 특징

- 각 프로세서는 자체 계산 능력을 가지며, 하나의 공통 기억 장소나 각자의 전용 기억 장소를 가질 수 있다.
- 전체 시스템은 하나의 통합된 운영체제 또는 각기 독립된 운영체제에 의해 관리된다.

#### Flynn의 분류 (명령어/데이터 흐름 기준)

- **SISD (Single Instruction, Single Data):** 단일 프로세서가 단일 데이터 흐름을 순차적으로 처리하는 전통적인 폰 노이만 구조.
- **SIMD (Single Instruction, Multiple Data):** 단일 명령어가 여러 처리 요소에서 각기 다른 데이터에 대해 동시에 실행되는 구조. (예: 벡터 프로세서, 배열 컴퓨터)
- **MISD (Multiple Instruction, Single Data):** 다수의 명령어가 단일 데이터 흐름에 대해 실행되는 구조로, 실제로는 거의 응용되지 않는다.
- **MIMD (Multiple Instruction, Multiple Data):** 여러 프로세서가 각자 다른 명령어와 다른 데이터를 처리하는 진정한 의미의 병렬 처리 구조.

#### 구조에 따른 분류

- **파이프라인 프로세서 (Pipeline Processor):** 하나의 프로세스를 여러 단계로 나누어, 각 단계가 동시에 다른 데이터를 처리하도록 하여 전체 작업 시간을 단축시키는 기법.
- **벡터 프로세서 (Vector Processor):** 벡터(배열) 데이터에 대한 연산을 하나의 벡터 명령어로 처리하여 루프 오버헤드를 줄이는 고성용 프로세서. 슈퍼컴퓨터에 주로 적용된다.
- **배열 프로세서 (Array Processor):** 하나의 제어 유니트(CU)가 다수의 처리 요소(PE)를 동기적으로 제어하는 SIMD 구조.

#### 결합도 및 연결 방식에 따른 분류

- **메모리 결합도:**
    - **강 결합 시스템 (Tightly-coupled):** 모든 프로세서가 하나의 메모리를 공유하며 통신하는 구조.
    - **약 결합 시스템 (Loosely-coupled):** 각 프로세서가 별도의 로컬 메모리를 가지며 메시지 전송 방식으로 통신하는 구조.
- **프로세서 연결 방식:**
    - **시분할 공유 버스:** 하나의 버스를 모든 장치가 공유. 경제적이지만 병목 현상 발생 가능.
    - **크로스바 교환 행렬:** 모든 프로세서가 모든 메모리에 동시에 접근 가능. 성능은 우수하나 비용이 높고 복잡함.
    - **하이퍼큐브:** N차원 큐브 형태로 노드를 연결하여 경제성과 빠른 통신 속도를 절충한 구조.

#### 운영체제 형태에 따른 분류

- **주/종 (Master/Slave) 구조:** 하나의 주(Master) 프로세서가 운영체제 및 입출력을 담당하고, 나머지 종(Slave) 프로세서는 연산만 수행하는 비대칭 구조.
- **분리 수행 (Separative Executive) 구조:** 각 프로세서가 독립적인 운영체제를 가지는 구조. 한 프로세서의 고장이 전체 시스템에 영향을 주지 않는다.
- **대칭적 처리 (Symmetric Treatment) 구조:** 모든 프로세서가 완전한 기능을 갖춘 하나의 운영체제를 공유하여 수행하는 가장 강력하고 복잡한 구조.

### 3.3. 분산 처리 시스템 상세 분석

#### 정의 및 목적

- 데이터베이스와 처리 장치가 지역적으로 분산되어 네트워크로 상호 결합된 시스템 환경.
- **목적:** 자원 공유, 연산 속도 및 신뢰성 향상, 컴퓨터 통신.

#### 장점 및 단점

- **장점:** 장치/데이터 공유 용이, 중앙 컴퓨터 과부하 감소, 시스템의 점진적 확장 가능.
- **단점:** 소프트웨어 개발이 복잡함, 보안 문제 발생 가능.

#### 투명성 (Transparency)

사용자가 분산된 자원을 마치 하나의 통합된 시스템처럼 인식하도록 하는 개념. 자원의 위치, 접근 방식, 복제, 고장 등을 사용자로부터 숨기는 것을 목표로 한다.

- **투명성의 종류:** 위치, 접근, 이주, 복제, 병행, 규모, 성능, 고장 투명성 등.

#### 분류

- **프로세스 모델에 따라:**
    - **클라이언트-서버 모델:** 자원을 요청하는 클라이언트와 제공하는 서버로 구성.
    - **프로세스 풀 모델:** 연산을 처리하는 프로세서 풀과 이를 사용하는 터미널로 구성.
    - **혼합 모델:** 위 두 모델을 결합한 형태.
- **위상(Topology)에 따라:**
    - **완전 연결, 부분 연결, 계층(트리), 성형(Star), 환형(Ring), 다중 접근 버스 구조** 등.
