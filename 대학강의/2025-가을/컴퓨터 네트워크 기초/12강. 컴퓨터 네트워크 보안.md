
본 문서는 컴퓨터 네트워크 보안의 핵심 원칙, 주요 위협 요소, 그리고 이를 방어하기 위한 핵심 기술을 종합적으로 분석한다. 정보 보안은 권한 있는 사용자만 정보에 접근하도록 보장하는 **기밀성**, 정보의 임의적 조작을 방지하는 **무결성**, 그리고 정당한 요구에 항상 서비스를 제공하는 **가용성**을 기본 요구사항으로 한다. 네트워크 시스템의 위험은 외부의 **위협**이 시스템 내부의 **취약점**을 이용하여 조직의 **자산**에 피해를 입힐 잠재적 가능성으로 정의된다.

주요 네트워크 프로토콜(TCP/IP, ARP 등)은 설계 당시 보안을 심도 있게 고려하지 않았기 때문에 본질적인 취약점을 내포하고 있으며, 공격자들은 이를 악용하여 **스니핑(도청)**, **스푸핑(변조)**, **서비스 거부(DoS) 공격** 등 다양한 공격을 수행한다. 이러한 위협에 대응하기 위해 전송 구간 데이터를 보호하는 **암호화 기술**이 필수적이다. 암호화 기술은 동일한 키를 사용하는 **대칭키**, 공개키/개인키 쌍을 사용하는 **비대칭키**, 그리고 이 둘의 장점을 결합한 **혼합 암호화 방식**으로 분류된다. 또한, 데이터의 무결성을 보장하기 위해 **해시 함수**와 **전자 서명** 기술이 활용된다.

실질적인 방어 체계는 **방화벽**, **침입 탐지 시스템(IDS)**, **웹 애플리케이션 방화벽(WAF)**과 같은 전문 보안 장비를 통해 구축된다. 더불어, **가상 사설망(VPN)** 기술은 공용 네트워크상에서 안전한 통신 채널을 제공하며, **네트워크 망분리**는 외부 위협으로부터 내부 핵심망을 물리적 또는 논리적으로 격리하여 보안 수준을 높이는 효과적인 전략이다.

## I. 컴퓨터 네트워크 보안의 기본 원칙

### 1. 정보 보안의 요구사항

정보 보안 활동은 관리적, 물리적, 기술적 관점에서 IT 인프라의 전 과정(설계, 구현, 테스트, 운영)에 걸쳐 다음의 핵심 요구사항을 충족시켜야 한다.

|   |   |   |
|---|---|---|
|요구사항|영문명|설명|
|**기밀성**|Confidentiality|권한이 있는 사람 혹은 시스템만이 관련 정보를 확인하거나 참조할 수 있도록 보장함.|
|**무결성**|Integrity|정보가 임의로 조작되거나 변경되지 않도록 보호함.|
|**가용성**|Availability|정당한 사용자의 서비스 요구에 대해 시스템이 항상 가용한 상태를 유지하고 서비스를 제공할 수 있도록 보장함.|
|**법적 준거성**|-|관련 법률, 제도, 규칙을 준수하여 시스템을 개발하고 관리해야 함.|

### 2. 정보 시스템 위험의 구성 요소

정보 시스템의 위험(Risk)은 외부의 위협이 내부의 취약점을 이용하여 자산에 피해를 입힐 수 있는 잠재적 가능성으로, 다음과 같은 요소로 구성된다.

|   |   |
|---|---|
|구분|설명|
|**자산 (Assets)**|조직이 보호해야 할 대상. 정보, 하드웨어, 소프트웨어, 시설, 인력, 기업 이미지 등이 포함됨.|
|**위협 (Threats)**|자산에 손실을 초래할 수 있는 원치 않는 사건의 잠재적 원인 또는 행위자.|
|**취약성 (Vulnerability)**|위협에 의해 발생할 수 있는 정보 보안상의 모든 허점. 사용자의 권한 이상의 동작을 허용하거나 사회공학적 기법에 의한 약점 등을 포함함.|

### 3. 네트워크 보안의 정의

네트워크 보안은 허가되지 않은 접근, 실수, 재해, 해킹 공격 등 네트워크 기반 기술의 취약점을 악용한 공격으로부터 네트워크 및 네트워크 기반 정보자산을 보호하기 위한 일련의 활동을 의미한다.

## II. 주요 네트워크 프로토콜 취약점 및 공격 기법

### 1. 프로토콜별 취약점

|   |   |   |
|---|---|---|
|프로토콜|취약점 설명|주요 공격 기법|
|**TCP/UDP**|TCP의 Control Flag와 Sequence Number 제어 가능성, UDP의 확인 및 응답 절차 부재를 이용한 공격에 노출됨.|포트 스캐닝, DoS, 세션 탈취, 보안 솔루션 우회|
|**IP/ARP**|IP는 보안을 고려하지 않고 설계되어 헤더 필드 조작이 용이함. ARP는 요청 및 응답 시 인증 절차가 없어 변조 공격에 매우 취약함.|IP 스푸핑 (인증 우회, DoS), ARP 스푸핑, ARP 캐시 포이즈닝|
|**응용 프로토콜 (HTTP 등)**|TCP/IP 기반의 공개된 프로토콜로, 프로토콜의 특징을 악용한 공격이 가능하며, 응용 소프트웨어 개발 시 취약점이 유입될 수 있음.|HTTP Get Flooding, Slowloris 공격|

### 2. 주요 공격 기법 분석

#### 포트 스캐닝 (Port Scanning)

시스템에서 활성화된 서비스(열려 있는 포트)를 확인하는 기법으로, 공격의 사전 정찰 단계에서 주로 사용된다.

|   |   |
|---|---|
|스캐닝 방식|설명|
|**TCP Scanning**|정상적인 3-way handshaking 연결 과정을 모두 수행하여 포트의 활성화 여부를 확인.|
|**SYN Scanning**|3-way handshaking을 완료하지 않고, 서버로부터 SYN-ACK 응답을 받는지만 확인. 서버에 로그가 남지 않는 은닉 스캐닝(stealth-scanning) 방식 중 하나.|
|**UDP Scanning**|UDP 패킷 전송 후 'ICMP unreachable' 메시지 수신 여부를 통해 포트의 비활성화 상태를 판단.|
|**Stealth Scanning**|TCP 헤더의 FIN, NULL, ACK 플래그 등을 비정상적으로 조작하여 패킷을 보낸 후, 그 응답을 통해 포트 상태를 확인함. 모든 플래그를 설정하는 방식은 XMAS 스캔이라 부름.|

#### 스니핑 (Sniffing)

소프트웨어나 하드웨어(T-Sensor)를 이용해 네트워크를 통해 전송되는 데이터를 도청하는 해킹 기법이다.

|   |   |
|---|---|
|스니핑 방식|설명|
|**Promiscuous Mode**|네트워크 인터페이스를 '무작위 모드'로 설정하여, 자신의 MAC 주소가 아닌 다른 목적지로 향하는 프레임까지 모두 수신함. `tcpdump`, `Wireshark`와 같은 도구를 이용해 패킷을 분석함.|
|**Proxy Application**|`BurpSuite`, `Paros`와 같은 프록시 프로그램을 설치하고 클라이언트(예: 웹 브라우저)의 프록시 설정을 변경하여 클라이언트와 서버 간의 통신 데이터를 가로채고 분석함.|

#### 스푸핑 (Spoofing)

IP 주소, MAC 주소 등 시스템의 고유한 식별 정보를 위조하여 공격 대상을 속이는 해킹 기법이다.

|   |   |
|---|---|
|스푸핑 종류|설명|
|**ARP Spoofing**|ARP 프로토콜의 인증 부재 취약점을 악용, 변조된 ARP Reply를 지속적으로 전송하여 로컬 네트워크 내 호스트들의 ARP 캐시 테이블을 오염시킴. 이를 통해 특정 호스트의 트래픽을 공격자에게로 유도하여 2차 공격(정보 탈취 등)을 수행함.|
|**DNS Spoofing**|정상적인 DNS 질의에 대해 변조된 IP 주소를 응답하여 사용자가 악의적인 사이트로 접속하도록 유도하거나, 악성코드를 통해 시스템의 `hosts` 파일을 직접 변조함.|
|**IP Spoofing**|패킷의 출발지 IP 주소를 다른 주소로 위조하여 IP 기반 인증을 무력화하거나 공격자의 신원을 은폐함.|
|**E-Mail Spoofing**|이메일의 발신자 주소를 위장하여 메일을 발송함.|

#### 서비스 거부 공격 (DoS/DDoS)

대량의 접속 요청이나 비정상적인 트래픽을 발생시켜 대상 시스템의 자원을 고갈시키고, 정상적인 서비스를 제공하지 못하도록 만드는 공격이다. 분산 서비스 거부 공격(DDoS)은 다수의 시스템을 동원하여 공격하는 방식이다.

|   |   |
|---|---|
|공격 종류|설명|
|**TCP SYN Flooding**|3-way handshaking의 마지막 단계인 ACK를 보내지 않아 서버가 SYN_RECV 상태에서 대기하도록 만듦. 이 과정을 반복하여 서버의 TCP 세션 풀(Session Pool)을 고갈시킴.|
|**Land Attack**|출발지 IP와 목적지 IP를 모두 공격 대상(Victim)의 IP로 설정한 패킷을 전송함. 패킷을 수신한 시스템은 자기 자신에게 응답을 보내는 루프에 빠져 부하가 가중됨.|
|**Ping of Death**|MTU(최대 전송 단위) 크기를 초과하는 매우 큰 ICMP 패킷을 전송하여, 수신 시스템에서 과도한 패킷 단편화 및 재조립 과정이 발생하도록 유도해 시스템 장애를 일으킴.|
|**DrDoS**|IP 스푸핑과 반사체(Reflector) 서버를 이용한 DDoS 공격 기법. 공격자는 출발지 IP를 공격 대상의 IP로 위조하여 다수의 서버에 요청을 보내고, 서버들은 위조된 IP 주소(공격 대상)로 대량의 응답을 보내게 됨.|

## III. 전송 구간 데이터 보호를 위한 암호화 기술

### 1. 암호화 기본 개념

암호화는 권한 있는 주체만 내용을 알 수 있도록 평문을 암호문으로 변환하는 기법이다.

- **양방향 암호화**: 암호화와 복호화를 모두 지원.
- **단방향 암호화**: 암호화만 가능하며, 수학적으로 복호화가 불가능.

### 2. 핵심 암호화 기법

#### 대칭키 및 비대칭키 암호화 비교

|   |   |   |
|---|---|---|
|구분|대칭키(비밀키) 암호화|비대칭키(공개키) 암호화|
|**키 구조**|암호화와 복호화에 동일한 키 사용|공개키(암호화/검증)와 개인키(복호화/서명)로 구성된 키 쌍 사용|
|**장점**|알고리즘이 가볍고 암/복호화 속도가 빠름|키 분배 및 관리가 용이하며, 안전한 키 교환이 가능|
|**단점**|키 교환 및 관리가 어려움 (참가자 수 증가 시 키 개수 급증)|암/복호화 속도가 상대적으로 느림|
|**주요 알고리즘**|DES, SEED, AES, ARIA, LEA|RSA, ECC|

#### 혼합 암호화 방식

대칭키와 비대칭키 방식의 장점을 결합한 방식. 실제 데이터는 속도가 빠른 **대칭키**로 암호화하고, 데이터 암호화에 사용된 대칭키는 안전한 키 교환이 가능한 **비대칭키(공개키)** 방식으로 암호화하여 상대방에게 전달한다.

#### 해시 함수 (Hash Functions)

임의 길이의 입력 데이터로부터 고정된 길이의 해시 값(메시지 다이제스트, MD)을 출력하는 단방향 함수로, 데이터 무결성 검증과 인증에 활용된다.

|   |   |
|---|---|
|특징|설명|
|**단방향성 (One-Way)**|해시 결과값으로부터 원본 입력값을 계산하는 것이 수학적으로 불가능함.|
|**충돌 저항성 (Collision Resistance)**|동일한 해시 값을 갖는 서로 다른 두 개의 입력값을 찾는 것이 불가능함.|
|**고정 길이 출력**|입력값의 길이에 상관없이 항상 고정된 길이의 결과값을 출력함.|

### 3. 전자 서명 (Digital Signatures)

전자문서 작성자의 신원을 확인하고, 해당 문서가 위조되지 않았음을 증명하는 전자 형태의 서명이다.

- **요건**: 위조 불가, 서명자 인증, 부인 방지, 분쟁 해결 지원.
- **생성 및 검증 과정**:
    1. **생성**: 원본 전자문서의 해시 값(Message Digest)을 계산한 후, 서명자의 **개인키**로 암호화하여 전자서명을 생성한다.
    2. **검증**: 수신자는 서명자의 **공개키**로 전자서명을 복호화하여 원본 해시 값을 얻는다. 동시에 수신한 전자문서로부터 직접 해시 값을 계산하여 두 값이 일치하는지 비교함으로써 서명의 유효성과 문서의 무결성을 검증한다.

### 4. 가상 사설망 (VPN - Virtual Private Network)

공용 네트워크(인터넷)를 통해 전용선과 같은 보안성과 정책을 보장하는 암호화된 연결(터널링)을 제공하는 응용 기술이다.

- **주요 기술 유형**:
    - **IPSec**: IP 패킷 단위까지 암호화하는 터널링 방식.
    - **SSL/TLS VPN**: 웹 브라우저 기반으로 편리하게 접근할 수 있는 방식.
    - **MPLS VPN**: Multi-Protocol Label Switching 기술 기반의 VPN.

## IV. 네트워크 보안 솔루션 및 장비

### 1. 방화벽 (Firewall)

내부 네트워크와 정보자산을 보호하기 위해, 사전에 정의된 규칙에 따라 허가된 패킷(연결)만 통과시키고 나머지는 차단하는 보안 솔루션이다.

|   |   |
|---|---|
|주요 기능|설명|
|**Access Control**|패킷 필터링 또는 프록시 방식을 통해 네트워크 접근을 제어하는 기본 기능.|
|**Logging & Audit**|방화벽을 통과하는 패킷 및 연결 정보를 기록하여 비정상적인 접속 시도를 추적하고 분석함.|
|**Authentication**|사용자 인증(패스워드, OTP 등) 및 접속 호스트의 정당성을 확인하여 신뢰성을 보장함.|
|**Data Encryption**|VPN 기능을 제공하여 방화벽 간 데이터 전송 시 통신 내용을 암호화함.|
|_최근에는 방화벽, 바이러스 차단, 공격자 차단 등 다양한 보안 기능을 통합한_ _**UTM(통합 위협 관리)**_ _장비로 발전하고 있다._||

### 2. 침입 탐지 및 방지 시스템 (IDS/IPS)

네트워크 및 서비스에 대한 공격을 실시간으로 탐지하는 시스템(IDS)에서 한 단계 더 나아가, 탐지된 공격을 능동적으로 차단하는 기능까지 포함한 것이 침입 방지 시스템(IPS)이다.

- **운영 방식**:
    - **호스트 기반 (HIDS)**: 개별 서버에 직접 설치하여 시스템 내부의 다양한 정보를 이용해 침입을 탐지.
    - **네트워크 기반 (NIDS)**: 네트워크 특정 구간(세그먼트)에 설치되어 해당 구간을 통과하는 트래픽을 감시.
- **탐지 방식**:
    - **오용 탐지 (Misuse Detection)**: 이미 알려진 공격 패턴(시그니처)을 DB에 등록하고, 트래픽과 비교하여 일치하는 경우 공격으로 탐지. 오탐율이 낮지만 신규 공격에는 취약함.
    - **이상 탐지 (Anomaly Detection)**: 시스템의 정상적인 상태 패턴을 학습한 후, 이와 다른 비정상적인 패턴이 발생하면 공격으로 판단. 새로운 공격에 대응할 수 있으나 정상적인 행위를 오탐할 가능성이 있음.

### 3. 웹 애플리케이션 방화벽 (WAF - Web Application Firewall)

OSI 7계층(응용 계층)에서 동작하며, SQL Injection, Cross-Site Scripting(XSS) 등 웹 기반 서비스에 특화된 공격을 탐지하고 차단하는 전문 보안 솔루션이다.

### 4. 네트워크 망분리 (Network Segmentation)

보안 수준이 다른 네트워크를 분리하여 운영하는 방법으로, 일반적으로 외부 인터넷망과 내부 업무망을 분리하여 외부의 위협이 내부망으로 직접 전파되는 것을 방지한다. 분리 방식은 물리적, 논리적, 하이브리드 방식으로 구분된다.
