###  Swap 공간이란?

**Swap 공간(Swap Space)**은 운영체제에서 **주기억장치(RAM)가 부족할 때 보조기억장치(디스크)의 일부를 마치 메모리처럼 사용하는 영역**을 말합니다. 쉽게 말해, **메모리의 확장판** 역할을 하는 디스크 공간이에요.

---

###  주요 특징
- **보조기억장치 활용**  
  RAM이 부족하면, 사용하지 않는 메모리 페이지를 디스크의 Swap 공간으로 옮겨 놓습니다.
  
- **가상기억장치의 일부**  
  Swap 공간은 가상기억장치(Virtual Memory) 관리 기법의 핵심 요소 중 하나입니다.
  
- **동작 방식**  
  - 프로세스 실행 중 메모리가 부족하면 → 일부 데이터를 Swap 공간으로 내보냄 (Swap Out)  
  - 다시 필요해지면 → Swap 공간에서 RAM으로 불러옴 (Swap In)

---

###  예시
- RAM이 8GB인데 실행 중인 프로그램들이 총 10GB 메모리를 필요로 한다면,  
  부족한 2GB를 디스크의 Swap 공간에 저장해 두고 필요할 때 불러옵니다.

---

###  장점과 단점
| 장점 | 단점 |
|------|------|
| RAM보다 큰 프로그램 실행 가능 | 디스크 I/O 속도가 RAM보다 훨씬 느려 성능 저하 |
| 시스템 안정성 확보 | Swap이 과도하면 쓰레싱(Thrashing) 발생 |
| 메모리 부족 시 응급 대처 가능 | SSD 수명 단축 가능 (잦은 쓰기/읽기) |

---

 정리하면, **Swap 공간은 RAM이 부족할 때 디스크를 임시 메모리처럼 활용하는 영역**으로, 시스템이 멈추지 않고 계속 동작할 수 있게 해주는 안전장치입니다.
