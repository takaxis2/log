## 세그멘테이션 (Segmentation)

- **개념**: 프로그램을 의미 단위(세그먼트)로 나누어 메모리에 적재하는 방식
    
- **세그먼트 예시**: 함수, 배열, 데이터 구조, 코드 블록 등
    
- **특징**:
    - 세그먼트 크기는 **가변적** (고정되지 않음)
        
    - 논리적 구조를 반영 → 프로그래머 관점에서 직관적
        
    - 외부 단편화(External Fragmentation)가 발생할 수 있음
        
- **장점**: 프로그램 구조와 잘 맞아 관리가 용이
    
- **단점**: 외부 단편화 문제, 관리 오버헤드 증가
    

## 페이징 (Paging)

- **개념**: 프로그램을 **고정된 크기**의 페이지 단위로 나누어 메모리에 적재하는 방식
    
- **페이지 크기**: 보통 4KB, 8KB 등 일정한 크기
    
- **특징**:
    - 논리적 주소 공간을 페이지 단위로 나눔
        
    - 물리적 메모리는 프레임(Frame) 단위로 나눔
        
    - 페이지 테이블을 통해 논리 주소 ↔ 물리 주소 매핑
        
    - 내부 단편화(Internal Fragmentation)가 발생할 수 있음
        
- **장점**: 외부 단편화 없음, 메모리 활용 효율적
    
- **단점**: 페이지 테이블 관리 비용, 마지막 페이지에서 내부 단편화 발생 가능
    

## 혼용 기법 (Segmentation + Paging)

- **개념**: 세그멘테이션과 페이징을 결합한 방식
    
- **방법**:
    - 프로그램을 먼저 **세그먼트 단위**로 나눔
        
    - 각 세그먼트를 다시 **페이지 단위**로 나눔
        
- **특징**:
    - 세그먼트 테이블과 페이지 테이블을 모두 사용
        
    - 세그먼트의 논리적 의미를 유지하면서도 페이징의 장점을 활용
        
- **장점**:
    - 외부 단편화 문제 해결 (페이징 덕분)
        
    - 프로그램 구조 반영 가능 (세그멘테이션 덕분)
        
- **단점**:
    - 관리 구조가 복잡해짐 (테이블 2개 필요)
        
    - 주소 변환 과정이 길어져 성능 저하 가능
